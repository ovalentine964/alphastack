## 1. Name Analysis — ALPHA STACK

### 1.1 Etymology & Meaning

| Component | Meaning | Trading Context |
|-----------|---------|-----------------|
| **ALPHA** | The first, the origin, the excess | Alpha = excess returns over a benchmark. The holy grail every fund manager chases. It signals performance, edge, and outperformance. |
| **STACK** | Layered, accumulated, compounding | A technology stack — layers of intelligence (data, signals, execution, risk) stacked on top of each other. Also slang for building wealth. |

**Combined meaning:** A system that **stacks multiple layers of intelligence** — data ingestion, signal generation, risk management, execution — into a single compounding engine that generates **alpha** (excess returns) for the user.

> "Every edge you have stacks on top of the last one. Alpha Stack doesn't give you one advantage — it gives you a stack of them. Data intelligence, pattern recognition, risk controls, and execution — all layered, all compounding, all working for you."

## 2. Brand Identity

### 2.1 Core Values

| Value | What It Means for ALPHA STACK |
|-------|-------------------------------|
| **Intelligence** | Every feature is backed by data, logic, and smart design. No gimmicks, no gambling. |
| **Accessibility** | Institutional-grade tools made available to everyday traders. No gatekeeping. |
| **Reliability** | The system works when you need it. Uptime, accuracy, and consistency are non-negotiable. |
| **Transparency** | Users see what the system sees. No black boxes. Performance is auditable. |

### 2.2 Target Audience

**Primary:** African retail traders  
- Age: 22–40
- Tech-savvy, mobile-first
- Interested in forex, crypto, and equities
- Looking for tools that level the playing field against institutional players
- Value affordability and simplicity without sacrificing power

**Secondary (Global Expansion):**
- Retail traders in emerging markets (Southeast Asia, Latin America, Middle East)
- Diaspora communities seeking trusted African-built fintech
- Independent traders who want algorithmic edge without coding

### 2.3 Brand Personality

| Trait | Description | Do | Don't |
|-------|-------------|-----|-------|
| **Smart** | Data-driven, analytical, precise | Use clean data visualizations, show logic | Overcomplicate, use jargon for jargon's sake |
| **Confident** | Bold claims backed by evidence | Make clear performance statements | Be arrogant, promise guaranteed returns |
| **Approachable** | Friendly, inclusive, not intimidating | Use plain language, warm UX copy | Be condescending, oversimplify |
| **Futuristic** | Forward-looking, innovative, AI-native | Use modern design, reference AI/ML naturally | Be gimmicky, use buzzwords without substance |

### 2.4 Brand Voice

- **Tone:** Confident but not arrogant. Smart but not condescending. We explain, we don't lecture.
- **Perspective:** "We built this for traders like us." First-person plural, peer-to-peer.
- **Language:** Clear, direct, active voice. Short sentences. Trading terms used naturally, never as flex.

**Example copy:**
> "Alpha Stack scans 200+ instruments in real time. You get the signals. You make the call. We handle the math."

## 4. Logo Concept — Desktop App

### 4.1 Primary Concept: "Stacked Alpha"

**Description:**  
A stylized letter **"A"** constructed from **three horizontal layers** (stacks) that subtly converge upward into an arrow/peak shape. The layers are slightly offset to create a sense of depth and motion. The bottom layer is the widest (data/foundation), the middle is narrower (intelligence), and the top converges to a sharp point (alpha/edge).

**Key elements:**
- The "A" shape is implied, not literal — geometric and abstract
- Three stacked layers with slight 3D depth (subtle shadow between layers)
- Upward momentum — the stack rises, suggesting growth and outperformance
- A small data-point or dot accent on the top layer (representing the signal/alpha)

**Color:**  
- Layers: gradient from Electric Blue (bottom) to Neon Green (top)
- Background: Deep Navy or transparent (adapts to light/dark mode)

### 4.2 Icon Sizes & Adaptations

| Size | Context | Adaptation |
|------|---------|------------|
| **512×512** | App icon, splash screen | Full detail — all three layers visible, gradient, subtle shadow |
| **256×256** | High-res displays | Full detail maintained |
| **128×128** | Standard app icon | Slightly simplified, shadows reduced |
| **64×64** | Dock/taskbar | Bold layers, minimal shadow, high contrast |
| **48×48** | Small taskbar | Simplified to essential shape |
| **32×32** | Favicons, system tray | Two layers only, arrow peak emphasized |
| **16×16** | Minimal favicon | Single bold "A" peak shape, solid color |

### 4.3 Alternative Concept: "Hex Stack"

**Description:**  
A hexagonal shape (tech/data feel) with internal layers creating an upward-pointing arrow pattern. More abstract, less letter-dependent.

**When to use:** If the "A" concept feels too literal or doesn't scale well.

### 4.4 Usage Rules

- **Minimum clear space:** Equal to the height of one "layer" on all sides
- **Never** stretch, rotate, or add effects beyond what's defined
- **Never** place on busy backgrounds without a backing shape
- **Always** use the approved color variations (full color, monochrome white, monochrome navy)

## 5. Logo Concept — Mobile App

### 5.1 Design Philosophy

Mobile icons must be **simpler** than desktop. At 48×48, fine details are invisible. The mobile icon should be **instantly recognizable** as a bold, abstract shape.

### 5.2 Primary Concept: "The Stack"

**Description:**  
Three **rounded-rectangle layers** stacked vertically with slight offset, forming an abstract upward arrow. The bottom layer is the widest, each subsequent layer is narrower, culminating in a pointed top.

**Key differences from desktop:**
- **Rounded corners** (iOS/Android convention)
- **No fine detail** — bold, solid shapes only
- **Stronger contrast** — works on any background
- **Squircle mask** — designed to look good inside iOS/Android icon shapes

### 5.3 Icon Sizes & Requirements

| Size | Context | Notes |
|------|---------|-------|
| **1024×1024** | App Store / Play Store listing | Full resolution master |
| **180×180** | iOS home screen (@3x) | Crisp at this size |
| **120×120** | iOS home screen (@2x) | Primary test size |
| **192×192** | Android adaptive icon | With safe zone padding |
| **48×48** | Android notification/small | Must remain recognizable |

### 5.4 Color Adaptations

- **Light mode icon:** Deep Navy background, Blue-to-Green gradient stack
- **Dark mode icon:** Slightly lighter navy background, same stack colors
- **Adaptive (Android):** Stack shape as foreground layer, Deep Navy as background

### 5.5 Mobile Icon Rules

- Must pass the **"squint test"** — recognizable when squinting at arm's length
- No text on the icon (the app name appears below)
- No photorealistic elements
- Maintain **consistent corner radius** with platform conventions

## 6. Visual Identity Guidelines

### 6.1 Typography

| Role | Font | Fallback | Usage |
|------|------|----------|-------|
| **Display / Headlines** | **Inter** (Bold, Extra Bold) | system-ui, sans-serif | App name, hero text, section headers |
| **Body / UI** | **Inter** (Regular, Medium) | system-ui, sans-serif | All UI text, descriptions, labels |
| **Monospace / Data** | **JetBrains Mono** | Consolas, monospace | Prices, numbers, code, terminal output |
| **Accent / Logo** | Custom wordmark (Inter-based) | — | "ALPHA STACK" logotype only |

**Why Inter:** Free, open-source, exceptional readability at all sizes, designed for screens, excellent number rendering (critical for trading), wide language support.

**Why JetBrains Mono:** Distinct characters (no 0/O confusion), ligature support, excellent at small sizes for dense data displays.

### 6.2 Typography Scale

| Level | Size (desktop) | Size (mobile) | Weight | Line Height |
|-------|---------------|---------------|--------|-------------|
| H1 | 32px | 24px | Extra Bold (800) | 1.2 |
| H2 | 24px | 20px | Bold (700) | 1.3 |
| H3 | 20px | 17px | Semi Bold (600) | 1.3 |
| Body | 14px | 15px | Regular (400) | 1.5 |
| Caption | 12px | 12px | Regular (400) | 1.4 |
| Data | 14px | 14px | Medium (500) | 1.3 |

### 6.3 Icon Style

| Aspect | Guideline |
|--------|-----------|
| **Style** | Flat with subtle depth — no heavy 3D, no skeuomorphism |
| **Line weight** | 1.5–2px for outline icons, consistent across set |
| **Corner radius** | 4px for small elements, 8px for cards, 12px for modals |
| **Shadows** | Subtle, low-opacity (`0 2px 8px rgba(0,0,0,0.15)`) — never harsh |
| **Icon set** | Phosphor Icons or custom-drawn in consistent style |

### 6.4 Consistency Across Platforms

| Platform | Key Considerations |
|----------|-------------------|
| **Desktop (Windows/macOS/Linux)** | System tray icon must work at 16px. Title bar uses wordmark. Splash screen uses full logo. |
| **Mobile (iOS)** | Follow HIG — rounded icon, SF Pro for system text, native navigation patterns. |
| **Mobile (Android)** | Material Design 3 conventions, adaptive icon with foreground/background layers. |
| **Web** | Responsive typography, dark mode default, logo adapts to header size. |
| **Telegram/Discord Bot** | Avatar = simplified icon (single stack shape). Inline embeds use wordmark. |
