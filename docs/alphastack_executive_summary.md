## 3. Alpha Strategy — Enhanced AlphaStack Flow

### 3.1 The 16-Step Enhanced AlphaStack (Volatility-Modified Propulsion Model) Flow

Alpha Stack's strategy engine follows a structured 16-step flow that combines academic rigor with practical execution:

| Phase | Steps | Description |
|-------|-------|-------------|
| **1. Data Perception** | 1–3 | Ingest market data (OHLCV, order book, sentiment, macro), normalize across sources, detect gaps/outliers |
| **2. Regime Detection** | 4–5 | Classify market regime (trending/ranging/volatile/crisis) using Markov switching models and volatility clustering |
| **3. Signal Generation** | 6–8 | Multi-timeframe technical analysis, fundamental scoring, sentiment aggregation → composite signal with confidence |
| **4. Alpha Verification** | 9–10 | Statistical significance testing (is this signal better than random?), edge decay monitoring |
| **5. Risk Budgeting** | 11–12 | Position sizing via modified Kelly Criterion, correlation-aware exposure limits, drawdown budget allocation |
| **6. Execution** | 13–14 | Smart order routing, TWAP/VWAP for larger sizes, slippage monitoring, fill quality tracking |
| **7. Review & Adaptation** | 15–16 | Post-trade analysis, strategy parameter updates, lesson extraction, memory consolidation |

### 3.2 Loop Architecture

Each phase uses appropriate reasoning loops:

| Loop Type | Application | Latency |
|-----------|------------|---------|
| **Reactive** (single-pass) | Tick-level price monitoring | <100ms |
| **Reflection** (self-correction) | Pre-trade analysis review | 2–5s |
| **Deliberation** (multi-option weighing) | High-conviction trade decisions | 5–30s |
| **Plan-and-Execute** | Portfolio-level allocation | Minutes |
| **Human-in-the-Loop** | Trades exceeding risk thresholds | Variable |

**Key insight:** Reflection loops reduce hallucination rates by 30–40% in financial analysis tasks, making them essential for any agent that generates trade recommendations.

### 3.3 Multi-Timeframe Analysis

AI simultaneously analyzes M1, M5, M15, H1, H4, D1, W1, and MN timeframes. The system uses a **weighted consensus model**:

- Higher timeframes carry more weight (D1 > H4 > H1 > M15)
- Confluence zones where multiple timeframes align receive highest confidence scores
- Conflicts between timeframes are flagged for human review or deferred execution

## 6. Multi-Agent System Design

### 6.1 Agent Architecture

Alpha Stack uses a **three-tier agent hierarchy** modeled on OpenClaw's gateway architecture with Hermes's closed learning loop:

```
DEPTH 0: COORDINATOR (main agent)
  ├── Receives user directives, market events
  ├── Decides which specialist to activate
  └── Synthesizes results, delivers reports

DEPTH 1: SPECIALIST AGENTS
  ├── Market Scanner — "Scan all instruments, identify opportunities"
  ├── Strategy Executor — "Execute strategy X on instrument Y"
  ├── Risk Manager — "Monitor portfolio risk, enforce limits"
  ├── News Analyst — "Process breaking news, score impact"
  └── Performance Auditor — "Review trades, update journal"

DEPTH 2: WORKERS (leaf tier — no autonomous decisions)
  ├── Data Fetcher — "Pull OHLCV for EUR/USD H4"
  ├── Indicator Calculator — "Run RSI/MACD/Bollinger on dataset"
  └── Order Placer — "Submit limit order to exchange"
```

### 6.2 Loop Assignments per Agent

| Agent | Loop Type | Frequency |
|-------|----------|-----------|
| Market Scanner | ReAct (search → read → synthesize) | On new events |
| Strategy Agent | Deliberation (weigh multiple signals) | Per signal |
| Risk Agent | Evaluation (accept/modify/reject) | Per trade |
| Execution Agent | Plan-and-Execute | Per trade |
| Monitor Agent | Event-driven polling | Continuous |
| Reflection Agent | Reflection loop | Daily/Weekly |

### 6.3 Memory System (Three-Layer)

| Layer | Purpose | Alpha Stack Equivalent |
|-------|---------|----------------------|
| **L1: Daily Notes** | Raw working context | Daily market data, signals, orders |
| **L2: Long-Term** | Curated facts & lessons | Trade journal, strategy memory, edge notes |
| **L3: Consolidation** | Periodic distillation | Weekly/monthly strategy review, parameter updates |

**The Closed Learning Loop:** After processing a Fed rate decision, the agent creates a reusable skill. Next time, it loads that skill and processes faster with better context. After 10 similar events, the skill is highly refined. **The system compounds knowledge.**

### 6.4 Framework Choice: LangGraph

**Recommended orchestration framework** for its graph-based control flow, persistent state management, and native human-in-the-loop interrupts — all critical for trading workflows.

## 11. Risk Management Framework

### 11.1 Position Sizing

**Fixed Percentage Risk (Recommended):** Risk 1–2% of account per trade.

```
Position Size = (Account Balance × Risk %) / (Entry Price - Stop Loss Price)
```

At $7 account, 1% risk = $0.14 per trade — perfectly viable for learning.

**Kelly Criterion** (for optimal sizing): Use half or quarter Kelly to reduce aggression.

### 11.2 Portfolio-Level Risk Controls

| Control | Description | Limit |
|---------|-------------|-------|
| Max positions | Concurrent open trades | 5–10 |
| Max exposure | Total notional vs. account | <5× (crypto), <20× (forex) |
| Max correlated exposure | Positions in correlated assets | <30% in correlated group |
| Daily loss limit | Stop trading threshold | 3–5% of account |
| Max drawdown circuit breaker | Halt all trading | 15–20% from peak |
| Per-exchange limit | Capital distribution | Split across 2–3 brokers |

### 11.3 Stop-Loss Strategies

| Type | Description | When to Use |
|------|-------------|-------------|
| Fixed % | Stop at X% below entry | Simple, beginners |
| ATR-based | Stop at N × ATR below entry | Adapts to volatility |
| Trailing stop | Stop follows price up, locks in profit | Trend following |
| Time-based | Exit if trade hasn't moved in X hours | Avoid capital tie-up |

### 11.4 Crypto-Specific Risks

| Risk | Mitigation |
|------|------------|
| Liquidation | Low leverage (<5×), wide stops |
| Exchange hack | Don't store >20% on any single exchange |
| Flash crash | Stop-market orders, not stop-limit |
| Funding rate manipulation | Monitor funding, reduce size before events |

### 11.5 Circuit Breakers

- **Bot-level:** Auto-pause if daily loss exceeds 3% of account
- **Strategy-level:** Disable strategy after 5 consecutive losses
- **System-level:** Flatten all positions if drawdown exceeds 20%
- **Human escalation:** Alert + require approval for trades exceeding 2% portfolio risk
